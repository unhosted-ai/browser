About

Why Delta Practice?

The name

Δ is the Greek letter for change — and the shape a river makes where it meets the sea. The agent in this browser exists to manage the delta between what you already know and what's on the page in front of you. Delta is also where things land: in a privacy-respecting browser, your activity is the river, the open web is the sea, and the delta is where attention stays — not where it gets carried out by trackers.

Practice is the second word, and the deliberate one. Software in practice, not in theory. A discipline of getting the mundane things right — the keyboard shortcuts, the address bar, the way windows manage focus — because an AI browser that treats those as incidental ends up being a chat panel with tabs. And practice as in: we're still figuring it out. The repo is public on day one because the practice happens in the open.

The thesis

The next browser is not a Chrome with a chat button. It's a browser that runs the model on your machine, reads pages with permission, acts on your behalf with consent, and forgets when you tell it to.

Each clause is load-bearing. Local LLM by default — every other AI browser ships your queries (and often the page) to a cloud. Permission lives in the runtime, not the prompt — a malicious page can't hijack the agent because the gate fires between the model's tool-call and the actual handler. Conversations persist on your disk; delete a thread and it's gone. There is no remote conversation log because there is no remote.

Open source — and why

The repo is public from day one. Three reasons:

  1. Privacy claims need verification. "Local-first, no telemetry, keys in the OS keychain" is a line on a landing page until someone can read the code. Closing the source on a privacy-first browser is a contradiction.
  2. Architecture is the moat, not the binary. The thoughtful parts of this project are the design docs and the discipline of staying true to them — not any clever trick. A team that wants to copy us would spend more time understanding the design than the code.
  3. Contributors over velocity. A solo project moves fast for two weeks and stops. A small community moves slower per week and doesn't stop. The repo is structured to be readable — you should be productive in a day.

The full version, including what we're explicitly not building and the constraints we accept, lives in apps/browser/docs/about.md.

Quickstart

Run it in three commands.

Requires pnpm and Node 20+. Tested on macOS; Linux works; Windows works in dev but the icon pipeline is mac-only for now.

shell
# clone, install, run
git clone https://github.com/Delta-Practice/Browser.git
cd Browser
pnpm install
pnpm delta:dev
shell
# bring up a local model — fastest path is Ollama
brew install ollama
ollama serve &
ollama pull llama3.2
# Delta auto-detects it within a few seconds

Already running LM Studio / llama.cpp / MLX? Start their local server and Delta picks them up. Cloud providers (OpenAI & OpenAI-compatible) are opt-in via Settings → API key, encrypted in your OS keychain.

What's in the box

Four pillars.

Privacy

Local by default

Auto-discovers Ollama, LM Studio, llama.cpp, MLX. Cloud providers are off until you explicitly enable them. No Delta account.

Agent

Reads & acts

The sidebar agent reads the active page (Phase 1 ✓). Read-tools, act-tools, and a per-origin permission model are next on the roadmap.

Identity

Device is the account

Profiles are local. API keys live in your OS keychain via safeStorage. The renderer never sees a key value.

Trust

Open-source

The repo is public for feedback. Brand assets are under brand/. Architecture & threat model are in apps/browser/docs/.

Architecture

One rule:
the agent never runs in the renderer.

All LLM calls, tool execution, and conversation state live in the main process. The renderer is a typed view over an event stream — it can request actions but never owns them. That keeps secrets out of the page-loading process and centralises tool authority where the permission gate can enforce it.

Privacy posture

What goes where.

You ↔ a website Cookies in the WebContentsView. Same as any browser.
You ↔ a local LLM HTTP to 127.0.0.1. Never leaves the machine.
You ↔ a cloud LLM Off by default. Opt-in via Settings, key encrypted in OS keychain.
You ↔ Delta Nothing. There is no Delta server.
Update check Once a day. Single signed manifest. No telemetry. Toggleable.
