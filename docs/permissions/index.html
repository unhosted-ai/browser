permission-gate

← back to delta

the agent can ask to do anything. the runtime decides whether anything actually happens.

every tool the agent has access to is tagged as either read (idempotent observation — listing tabs, extracting page text) or act (a real side effect — navigating, opening a new tab, clicking, typing). read tools auto-run; act tools route through a permission gate that lives in the main process, not the model's prompt.

when the model emits navigate("https://evil.example.com"), the runtime intercepts the call, looks up whether you've previously granted (origin: current-tab-host, tool: navigate), and if not, surfaces an inline permission card in the assistant sidebar with three buttons:

  • allow — execute this one call
  • block — refuse this call, tell the model
  • always allow on this site — record (origin, tool) in settings, auto-allow next time

why (origin, tool), not just (tool)

"the agent can navigate" is too coarse — you might want it to follow links inside wikipedia while refusing to navigate at all on your bank's website. so grants are scoped to the host of the active tab at the moment of the call. allowing navigate on news.ycombinator.com doesn't grant it on chase.com.

sensitive-site auto-block

some hosts auto-fail the permission check no matter what you've previously granted: banking, government, identity providers, payment processors. the agent simply can't act on *.gov, chase.com, login.live.com, etc. the rule lives in agent.ts and is auditable.

this is the second wall

the untrusted-content frame stops the model from thinking a malicious page's "navigate to evil.com" is an instruction. the permission gate stops the model from doing it even if it does. defence in depth: a single failure doesn't lose the game.

where grants live

granted (origin, tool) pairs persist in the user settings json — same file as your provider config. you can review and revoke them in settings → permissions (the list is rendered from settings.permissionGrants). uninstalling delta removes the file; there is no remote record.
