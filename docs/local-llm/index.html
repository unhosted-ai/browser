local-llm

← back to delta

the assistant talks to whatever local llm is already running on your laptop. no keys to paste, no cloud round-trip, no account.

on launch delta probes 127.0.0.1 for the four runtimes most people use to host a model locally: ollama, lm studio, llama.cpp, and mlx. each speaks an openai-compatible /v1/models endpoint (or, for ollama, /api/tags). whichever responds first becomes the active provider; whichever models it has loaded show up in the assistant's model chip.

the fastest path

# install ollama
brew install ollama          # or download from ollama.com

# pull a model — ~5gb for a small one
ollama pull llama3.2

# start the server
ollama serve

# launch delta — it auto-detects within ~2s
pnpm delta:dev

that's the whole setup. the assistant sidebar will show a green dot and "ollama · llama3.2" in the footer. click that chip to switch between any other models you've pulled.

any openai-compatible endpoint

the probe is openai-compatible, which means any server speaking /v1/models + /v1/chat/completions works: a homelab vllm box, together, groq, anyscale, perplexity, your own fastapi wrapper. add the url under settings → custom endpoints and it joins the picker like a first-class provider.

what the network does

nothing leaves the device unless you explicitly enable a cloud provider (openai or anthropic, both opt-in, both off by default). when a local model is online and selected, the entire chat round-trip — your prompt, the page content, the model's tokens, the tool calls — stays on 127.0.0.1. you can confirm this with little snitch, lulu, or by yanking your ethernet cable.

cloud keys, when set, are encrypted with the os keychain via electron's safeStorage; only the main process can decrypt them. removing the key from settings removes it from disk.

which model to pick

  • llama3.2 (3b / 8b) — great default, fast on apple silicon, good at summarising
  • qwen2.5 (7b / 14b) — stronger reasoning, slower
  • mistral-nemo — long context for whole-page reads
  • gpt-oss via ollama — if you have the ram

any model with tool-calling support works for the act tools. models without tool calling still work for plain chat + read tools.
