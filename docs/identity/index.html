a browser with no signup, no profile, no remote anything. you don't have a delta account because there's nothing for you to have an account on.

what lives where

everything delta knows about you lives under one directory:

~/Library/Application Support/Delta/   # macOS
~/.config/Delta/                       # Linux
%APPDATA%\Delta\                       # Windows

inside that folder, every file is plain json that you can open in any editor:

  • conversations/<uuid>.json — one file per chat thread
  • bookmarks.json — your saved urls
  • history.json — your visit history (capped fifo)
  • downloads.json — recent download metadata
  • settings.json — preferences, custom endpoints, permission grants
  • privacy.json — the rolling 30-day tracker counts

cloud keys are encrypted

the only sensitive thing inside settings.json is the API keys for cloud providers, if you enable them. those aren't stored plaintext — they're encrypted via electron's safeStorage, which uses the os keychain (macOS keychain, gnome-keyring/kwallet on linux, dpapi on windows). only delta's main process can decrypt them; even another process running as your user can't read the bytes off disk.

backups, sync, audit

because everything is flat json in a known directory, the boring tools work:

  • backup — time machine, restic, rsync. snapshot the folder.
  • sync across machines — point syncthing at the folder. now you have multi-device delta, owned by you.
  • audit — grep, jq, your editor. nothing is in a proprietary blob.
  • full reset — rm -rf ~/Library/Application\ Support/Delta and you're a fresh user.

the no-telemetry promise

delta makes zero outbound requests to delta infrastructure. there is no analytics endpoint, no crash reporter, no update server (the project doesn't even ship signed installers yet — see STATUS.md). every outbound request originates from a page you visited, a model you configured, or an api key you pasted. you can verify this with little snitch, lulu, or by yanking your network cable: a model running locally + a tab on localhost will function indefinitely offline.

app lock (optional)

settings → app lock turns on a touch id prompt before delta opens. it's off by default — the project's ethos is "device is the identity", and your already-logged-in os user is the boundary. but on a shared laptop or family machine, the lock gives you a second wall: even with the os logged in, opening delta requires your fingerprint. cancel and the app quits.

the prompt uses electron's systemPreferences.promptTouchID on macOS; nothing leaves the device, nothing is sent to delta. windows hello / linux polkit support is on the list.
